# 2019 年终总结

**2019-12-31** **20:47** **随笔** 

> 前言： 成长一直现在阶段的主要问题。在什么的平台，与什么样的人交往，坚持着什么样的方向，都只是为了自己的成长，以实现自己的梦想。

## 成长

成长, 这个问题会是这三年一直都在关注的问题。因为自己的目标凭借自己现在的能力尚且难以实现的。需要借助平台和他人来助力，或是学习更优秀的人，或是在更有挑战性的环境中角逐。

因此，在这个阶段，如果缺乏挑战或是无法学习，离开便是必然的。青春的时间总是短暂的，只有能够利用好这时间才能向自己的梦想更近一点，而不是只能羡慕别人。

但是并不是任何时候，都能有人一直帮助你，或指出你的问题所在，他们只能冷眼看着。

孤独始终伴随着成长。那其实你自己地成长只能通过自己的选择来完成。如果遇到良师益友，那便是可遇不可求的。可以如果没有也并不是需要伤感，一个人的成长也是充满了惊喜的，可能有时候你会突然意识到自己可以接受更多的挑战及面对更多的困难。

一切皆有可能，前提是你的坚持以及明白自己需要得到什么。

## 追求

为啥突然想起这个词语，大概是想到了一直在思考的性能优化的问题。想起了自己作为编程人员时，比较在意的代码风格的时候作过的一些事情：

- 删除/清理一些不符合规范的代码

- 向别人提出一些意见

- 吐槽自己曾经的代码垃圾之类的

...

便想到了其实我们应该有所追求，而不是顺其自然。不然在一些时候可能已经无力挽回。就如同我现在在维护的自己项目，总能有时候下意识的感觉到这样实现存在问题，但是往往功能可以实现，此时我并不会修正这个缺陷。而在日后，重新来阅读这些需要维护的代码的时候，总是可以感概一番。

显然，有些时候我们应该对自己严苛一些，让自己更有追求一些，而不是停留在功能实现即可，往后再改。

一直期待日后的自己，当下缺并无改变，往往都是一个糟糕的结局。

而当下我仍旧是一名编码人员，应该重视的问题便是自己对应代码、产品等地方的追求。浅层的实现，难以让你看到“世界”之大吧。同时也错过了无数精彩的时刻。

也因为只有追求才能更好的推动自己的成长。当然追求不仅是为了自己成长，同时也是减少对他人时间的浪费。

## 协调

> 有人的地方便有江湖。微笑，有人的地方就有矛盾才是。

其实协调的问题是不只是沟通的问题？

不是，只是沟通是解决问题的关键，而流程相较于沟通而言更为重要。因为协调是基于如何推动问题的解决，同时也因为是人，直接交流更为方便。

但是无论纸面上的沟通，或是口头上的沟通。实际上都存在着不少的问题。其中一个非常关键的就是`表达和效率`,而`表达`以纸面优胜，`效率`以口头优胜。但是两者都是存在一个关键的问题，信息容易被曲解或遗忘，以及另一个问题，协助人员之间的职能不一致，处理问题的时间协调困难的问题。

...

而上述的情况，只能靠解决问题的人员来推动问题的解决，并关注协调过程从沟通里面去提取有些的信息，寻找到能解决问题的人员和关键。

这将强烈依赖解决问题的人员的积极性。所以是`有人的地方便有江湖,有人的地方就有矛盾才是`。

因此如果我们希望解决协调的问题的关键将是`流程`, 一个能更高效的`沟通`的流程。而现在的我认为`流程`的实现的关键是：

- 解决人员

- 时间限制

- 核心信息

- 责任转移

- 问题等级

而这些关键核心作用如下：

- 时间限制是为了提高约束解决问题的人员的惰性。

- 责任转移是为了解决问题的人员判断问题的本质原因不在他管辖的范围之内，需要对应问题的解决人员参与进来，并希望由他来推动问题的解决。

- 问题等级则是为了提高解决问题的人员对应问题的关注度和意识问题的严重性，同时是否实时性的向问题发起人自动反馈问题的解决进度，以弥补时间限制的作用的缺陷。

- 核心信息是为了提醒问题的存疑点，解决问题的思考，问题所涉及的范围。这些信息的关键都将来着问题的提出人，以及解决人员之间沟通的内容。

是的，一切围绕问题本身来进行，由解决问题的人员来推动，提供便捷的手段来标记信息，以及及时的反馈机制。

当然上面已经提及过就效率而言，口头沟通的方式胜于纸面沟通，即邮件这样的方式。因此，`流程`的沟通方式将以实时聊天为主，引入 @import 这种形式的标记符来实现关键的标记。同时以邮件的方式，通知至解决问题的人员和参与人员。

选择相信`流程`来解决问题的效率，而非由所谓管理层模式来维护，同时认为依赖某人并不能真正的解决问题。关键在于选中不信任。

## 开源

2019 年的开源其实是一个个人兴趣的问题。从现在来看，并无惊喜。而是一种兴趣式的培养，希望有可以在这里寻找到兴趣相近的人，从开源项目中提取一些灵感来编写自己的项目。

而今年的主题是 Canvas / SVG ，基于它们之上来构建一些工具或者场景。而不是在局限于项目或者是业务，尽管有些时候灵感往来自于平时处理一些工作。但是更多的是你平时经历的事情，和为此的不经意的感慨，即生活。

今年其实蛮有意思的是，一向喜欢追逐新技术的我，并未跟随技术的潮流去学习，而是想自己为何在写代码，这些有何意义。

其实在不同的时刻，这个问题会有不一样的答案。

基于此我选择了记录自己思考的问题，阅读的文章 ...希望通过从中获取自己想要的灵感，并慢慢将它们转化为一个可行的项目。

这个过程便是将开源转为兴趣的工作。可能开源一个项目是需要责任的，但是在此之前更希望这是一个有意思的创意。等待合适的时刻，将它们上升为一个正式的项目。

感慨一下，目前我的 github 一言难尽。但是它慢慢的完善的，一个趋势。

现在仍需等待的便是 github's robot 的实现，来完成自动托管的任务。这可能才是开始......

## 方向

关于方向这个主题，其实应该在成长这个主题中讨论的。可是我回想了 2017 年和 2018 年自己考虑的一些成长问题和自己学习的技术，意识到其实成长和我的方向并无关系。

2017 年的时候，认为自己应该学习 Canvas / WebGL，成为一个计算机图形学的开发者。但遗憾是的，其实我只成为了一个前端工程师。

而 2018 年的时候，认为自己应该学习前端工程化，尽管也学习了，但我不能一直坚持在此路上走的更远。

这两年的经历一直都在让我成长，但是它们并无让我的方向变得清晰，我只是明白写着前端是因为兴趣，因为可以从它的持续变化中获得一些灵感，与喜欢的一些交互更近一些。

因此，方向无关成长。渴望的是在一个明确的方向上，能够清晰看见一些之前从未见过的精彩。

12 月的时候，心血来潮地了解了 Flutter，同时由于 Flutter 中无法使用 OpenGL 的技术，而学习了 Android。而在这个学习过程，所消耗的时间相较于两年前的学习时间而言，是非常短的。

如今，已经学习不少的技术，从 Java 到云计算，从云计算到前端，到现在的......

只明白，我的方向一直在变，最后我只是想拥有自己的作品。

而今年已经临近结束，在今年一路走下来，在一个新的平台上，理解少儿编程，从中得到了我们接下将与实现一系列的工具链为方向。

## 最后的杂谈

离 2019 年结束还有 4 个小时，现在的我还坐在公司里。也是在 31 号这一天写下这篇文章，在 31 号写下这篇文章之前我有想过该怎么去写更好一点。但是现在看来其实慢慢的在 31 号写完的时候，可以去想一些今年关键的地方。

> 通常我都是三年为一个周期，每一年都有一些感慨，而中间这一年往往都是平静的。

今年也正好是这样子，平静的一年，缺乏惊喜，在平凡中慢慢的成长。

在自己的感觉中，其实今年是成长最快的一年了，比之前任何时候都更为的明显，尽管依旧未能在今年结束的时候拿出一个自己感觉满意的作品。但是我感觉到思考的方向，正在向一个创意的方向上去，并从中实现了一些有趣的东西，也理解了当初思考的问题，正在慢慢将将其实现。也许这便是未来可期吧。

但是也很遗憾的是，同样告别了一些人。有些人只能陪你走一段路。

去年，无论是离别还是感情都是一个非常重要的组成部分。而今年，这些也有，但是我貌似依旧冷漠了，貌似懂了什么。因为知道自己想要什么，成长也在进行。

可能在以后回去看的时候，还能对的起自己吧。心怀罪恶。

总之，2019 年已经结束了。大概每一年，我在年终的时候并不喜欢讨论一些技术问题，总结自己学习了什么东西。而是想一下今年思考的问题，能否为将来带去一些他们可能遇到的情况的解决思路。

再见，2019 年！
